## TL;DR

The "Renaissance Edition" (Winter '26) introduces over 150 updates with a primary focus on making the developer platform AI-native and enabling Agentic Commerce . Key releases include the Dev MCP server supporting the full platform, the ability for Sidekick to generate apps and use app extensions, and a suite of tools for building commerce directly into AI agents . The session also covered quality-of-life updates like Admin Intents, regional performance metrics, and security enhancements .

---

## 1. The Renaissance Edition Overview

* **Theme**: The "Renaissance" theme symbolizes a modern era where AI is at the center of development .
* **Philosophy**: This is not just adding AI features to an existing platform; it is a platform designed for AI from the ground up .
* **Developer Impact**: Developers can now act as architects while AI agents handle scaffolding and implementation details .

## 2. AI-Native Dev Platform

* **Dev MCP Server**:
    * The Shopify Dev MCP (Model Context Protocol) server now supports the entirety of the dev platform .
    * Capabilities include generating GraphQL queries, scaffolding checkout and POS extensions, and assisting with Liquid and Hydrogen .
    * The server provides the AI with full context of the codebase to streamline building .
* **Dev Assistant Updates**:
    * The assistant on Shopify.dev now leverages the Dev MCP under the hood .
    * It utilizes a system prompt tailored for learning and discovery, highlighting relevant pathways rather than just generating code .
* **Full Stack Evolution**: These tools make it easier for specialized developers (e.g., theme devs) to expand into other areas (e.g., POS extensions), effectively making everyone a "Full Stack Shopify Developer" .

## 3. Agentic Commerce

* **Concept**: Developers can now build end-to-end commerce experiences within AI, from discovery to cart assembly and checkout .
* **The Tooling Suite**:
    * **Catalog API**: Enables agents to search billions of products (Universal Product IDs) across the ecosystem without needing individual merchant integrations . Includes a Product Lookup API for PDP details .
    * **Storefront MCP**: Handles store-specific interactions like return policies, shipping locations, and cart building. Every Shopify store exposes an MCP endpoint automatically .
    * **Checkout Kit**: Embeds checkout directly into AI experiences (Mobile and Web) while retaining merchant customizations and compliance .
* **Availability**: Storefront MCP and Checkout Kit for Mobile are available now; Catalog API and Checkout Kit for Web are coming in January .

## 4. Sidekick Updates

* **App Generation**: Sidekick can now generate simple apps directly in the admin based on merchant prompts .
* **Sidekick App Extensions**:
    * **Integration**: Apps can now integrate directly into Sidekick to expose data or invoke actions .
    * **Exposing Data**: Sidekick can query app data to answer merchant questions (e.g., performance metrics) .
    * **Invoking Actions**: Sidekick can trigger workflows via app intents (e.g., "create a return"), which the merchant then approves .
    * **Performance**: Limits are imposed for performance, aiming for a ~200ms response cutoff .

## 5. Building High-Quality Apps

* **Admin Intents API**:
    * Allows apps to open native Shopify editors (e.g., product or discount editors) directly within the app surface via a one-line API call .
    * Launched October 1st, supporting 13 resource types .
* **AppBridge Updates**:
    * **`app.extensions` API**: Allows developers to query which extensions are active on a shop (currently Checkout and Customer Accounts) to improve onboarding flows .
* **Performance Metrics**: Regional data has been added to admin performance metrics to help developers pinpoint geographical slowdowns .
* **Built for Shopify**: The grace period for meeting criteria (manual and automated) has been extended to 60 days .

## 6. Personal Favorites

* **Shop Minis**: Fully open public launch for all developers to build Shop Minis .
* **POS UI**: POS now leverages Polaris and Web Components, standardizing UI development across Admin, Checkout, and POS .
* **Metafields & Metaobjects**: Significant updates including increased limits per app (no longer a shared pool) and advanced filtering (numeric and date ranges) .
* **Security**: Introduction of expiring tokens for all OAuth grant types to enhance ecosystem security .
