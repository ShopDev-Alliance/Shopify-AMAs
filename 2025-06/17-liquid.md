## Expanded Key Takeaways

### 1. Horizon Theme and Theme Blocks Architecture

- Horizon is now the default theme for new merchants
- 10 versions available on the theme store
- Built from scratch to leverage theme blocks platform improvements
- Key features include:
  - Theme blocks with drag-and-drop functionality
  - Section and block categories
  - Conditional settings
  - Liquid doc integration
  - Static parameters for theme blocks
- Migration from Dawn was deemed too complex due to breaking changes

### 2. Static Parameters for Theme Blocks

- Enables passing props similar to React components
- Key use cases:
  - Product cards with inherited settings
  - Header/navigation components with context awareness
  - Index position tracking in loops (for_loop.index0)
- Separates developer concerns from merchant-visible settings
- Allows for more flexible component reuse across themes
- Future improvements planned for built-in parameters

### 3. AI Block Generation

- Available in any section supporting theme blocks (@theme in schema)
- Features:
  - Generates actual Liquid code in blocks folder
  - AI-generated blocks prefixed for easy identification
  - Code is editable and customizable post-generation
  - Right-click context menu for quick code editing
  - Hot reload functionality in editor
- Current limitations:
  - Block-level only (sections and templates coming)
  - No cleanup mechanism for unused blocks yet
  - Temporary storage until explicitly saved
- Future plans for section and template generation

### 4. Migration Strategies for Custom Themes

- Recommended approach:
  - Add new theme block-based sections alongside existing ones
  - Use section categories to mark legacy sections as "archive"
  - Implement custom section with generic blocks (group, text, image)
  - No forced migration for existing merchants
- Custom theme specific strategies:
  - Match schema between section blocks and theme blocks
  - Migrate JSON templates programmatically
  - Gradual transition without breaking changes
- Mixing section blocks and theme blocks not supported due to technical limitations

### 5. App Block Integration

- App blocks can be embedded within theme blocks
- Current limitations:
  - App blocks cannot nest other blocks (yet)
  - Product card support coming to Horizon
  - No drag-and-drop for overlaid elements
- Future improvements:
  - Parity between app blocks and theme blocks
  - Subfolder organization support
  - Better context sharing between blocks
  - Dynamic default settings based on merchant context

### 6. Performance and Limits

- 50 block per section limit being removed
- Performance improvements for theme blocks in progress
- Total capacity: up to 100 sections × blocks
- Considerations:
  - Editor performance with many blocks
  - Storefront rendering optimization
  - Balance between flexibility and performance
- Future focus on making blocks more performant

### 7. Developer Experience Improvements

- Design system best practices:
  - Keep blocks folder flat (subfolders coming)
  - Separate UI logic in snippets
  - Business logic in blocks
  - Smart vs dumb component pattern
- Private block naming conventions
- Double underscore not supported for differentiation
- Better template assignment workflows planned

2. Use static parameters for component reuse
3. Separate UI concerns into snippets
4. Keep block directories organized with naming conventions
5. Test AI-generated blocks before production use
6. Plan migration strategies for existing themes
7. Consider performance impact with many blocks
